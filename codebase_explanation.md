# Keka Timesheet Automation - Codebase Explanation

This document provides a comprehensive explanation of the purpose and working of all files within the `keka_automation` directory. Together, these files form an automation tool that fetches task definitions from Keka and automatically fills timesheets via Playwright.

---

## 🐍 Core Python Scripts

### 1. `fetch.py`
**Purpose:** Scrapes all available projects, phases, and tasks from Keka to create a reference mapping.
**Working:**
- Connects to an existing Chrome browser.
- Navigates through the "Add Time Entry" panel.
- Iterates through every project and expands every phase to discover all nested tasks.
- Saves the resulting structure into `fetched.json`.

### 2. `fill.py`
**Purpose:** The primary entry point for batch-filling timesheets based on an input file.
**Working:**
- Reads your worked tasks from `input.json`.
- Maps your task names to the official Keka project/task names using the data in `fetched.json` (with fuzzy matching support).
- Orchestrates the `KekaBot` to navigate to the correct weeks and populate entries.

### 3. `keka_bot.py`
**Purpose:** Wraps Playwright UI navigation into logical actions customized for Keka's DOM structure.
**Working:**
- Contains the `KekaBot` class.
- Handles calendar navigation and verifies timesheet slots.
- Exposes the complex `fill_entry` method which handles search, task selection, and time/comment entry.
- Implements robust fallback locators to handle UI variations.

### 4. `browser.py`
**Purpose:** Manages the connection between the automation suite and a local browser.
**Working:**
- Links Playwright to a **pre-existing, logged-in Chrome browser** via remote debugging (`localhost:9222`).
- This bypasses SSO and multi-factor authentication issues.

### 5. `utils.py`
**Purpose:** Shared utility functions (logging, environment loading, caching, and retries).

---

## 🛠️ Batch Scripts (Windows Ecosystem)

### 6. `launch_chrome.bat`
**Purpose:** Opens a dedicated Chrome window with remote debugging enabled.

### 7. `setup.bat`
**Purpose:** Initializes the virtual environment, installs dependencies, and prepares Playwright.

---

## 📄 Key Data Files

- **`input.json`**: Where you list what you worked on (date, task name, duration).
- **`fetched.json`**: A dictionary of all Keka projects/tasks (generated by `fetch.py`).
- **`.env`**: Stores your configurations (like browser paths).
- **`README.md`**: Detailed usage instructions and setup guide.
